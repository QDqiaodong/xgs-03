# 植物成长时间轴功能需求说明文档

## 一、功能概述

为植物档案详情页新增「成长时间轴」展示模块，以纵向时间轴的形式，将植物养护日志按**时间倒序**可视化呈现，直观展示植物从养护开始至今的完整成长轨迹与操作历程。

---

## 二、功能需求明细

| 序号 | 需求项 | 需求描述 | 完成状态 |
|:---:|:---|:---|:---:|
| 1 | 时间轴展示区 | 在植物档案详情页（备注区下方）新增独立的「🌱 成长时间轴」模块 | ✅ |
| 2 | 时间倒序排列 | 养护日志按时间从新到旧纵向排列；同一天内按具体创建时间（精确到分钟）倒序 | ✅ |
| 3 | 操作类型图标 | 左侧时间节点使用不同图标 + 颜色区分 5 种操作类型 | ✅ |
| 4 | 操作详情展示 | 右侧卡片展示：日期时间、操作类型标签、详情文字、长势/病害状态标签 | ✅ |
| 5 | 图片展示 | 支持展示养护日志关联的多张图片，网格布局，点击可全屏预览 | ✅ |
| 6 | 长内容展开/收起 | 详情文字超过 80 字默认折叠为 2 行，点击「展开/收起」切换 | ✅ |
| 7 | 滚动逐步加载 | 时间轴滚动到底部时自动加载下一页（每页 5 条），支持加载动画与「已加载全部」提示 | ✅ |

## 四、后端接口设计

### 4.1 新增分页查询接口

| 项目 | 内容 |
|:---|:---|
| **请求方式** | `GET` |
| **URL** | `/api/care-logs/plant/{plantArchiveId}/paged` |
| **路径参数** | `plantArchiveId` — 植物档案 ID（Long） |
| **Query 参数** | `page` — 页码（int，默认 0）；`size` — 每页条数（int，默认 5） |
| **返回类型** | `Page<CareLog>`（Spring Data 标准分页对象） |
| **排序规则** | 一级排序：`logDate DESC`（按记录日期倒序）；二级排序：`createdAt DESC`（按创建时间倒序，保证同一天内的精度） |

### 4.2 涉及的后端文件变更

| 文件 | 变更内容 |
|:---|:---|
| `backend/src/main/java/com/plantcare/repository/CareLogRepository.java` | 新增分页方法 `findByPlantArchiveIdOrderByLogDateDescCreatedAtDesc`，同步更新原有列表方法的排序规则 |
| `backend/src/main/java/com/plantcare/service/CareLogService.java` | 新增 `getPlantLogsPaged(Long plantArchiveId, int page, int size)`，同步更新 `getPlantLogs` 的排序 |
| `backend/src/main/java/com/plantcare/controller/CareLogController.java` | 新增 `GET /plant/{plantArchiveId}/paged` 端点 |

## 五、前端实现设计

### 5.1 新增组件

**`frontend/src/components/GrowthTimeline.vue`** — 独立的成长时间轴组件，可复用。

#### Props 定义

| Prop | 类型 | 必填 | 默认值 | 说明 |
|:---|:---|:---:|:---:|:---|
| `plantId` | `String \| Number` | ✅ | — | 植物档案 ID |
| `fetchLogs` | `Function` | ✅ | — | 分页数据获取函数，签名：`(plantId, page, size) => Promise<PageResult>` |
| `pageSize` | `Number` | ❌ | `5` | 每页加载条数 |

#### 暴露方法

| 方法 | 说明 |
|:---|:---|
| `resetAndLoad()` | 清空当前数据并从第 0 页重新加载（新增日志后调用） |

### 5.2 核心交互逻辑

#### 滚动加载（IntersectionObserver）
- 在时间轴底部放置哨兵元素（sentinel）
- 使用 `IntersectionObserver` 监听哨兵进入视口
- 进入视口时触发下一页加载，加载中禁用重复触发
- **关键修复**：将哨兵元素移出 `v-else` 条件块，并通过 `watch(logs)` + `nextTick()` 确保 DOM 更新后重新绑定 observer，避免初始为空时 observer 无法绑定的问题

#### 内容展开/收起
- 详情文字超过 80 字时显示「展开 ↓」按钮
- 使用 CSS `-webkit-line-clamp: 2` 实现 2 行截断
- 点击切换 `expandedItems Set`，控制是否移除截断样式

#### 图片预览
- 点击缩略图打开全屏黑色遮罩预览层
- 支持左右箭头翻页、关闭按钮、底部页码显示
- 打开时锁定 `body.overflow = hidden`，关闭时恢复

#### 入场动画
- 每条记录使用 `fadeInUp` 关键帧（opacity 0→1，translateY 20px→0）
- 按顺序延迟 `index * 0.05s`，形成瀑布流入场效果

### 5.3 集成的前端文件变更

| 文件 | 变更内容 |
|:---|:---|
| `frontend/src/api/index.js` | `careLogApi` 新增 `getByPlantPaged` 方法 |
| `frontend/src/views/ArchiveDetail.vue` | 替换原有养护日志列表为 `<GrowthTimeline>` 组件；新增日志后调用 `timelineRef.resetAndLoad()` 刷新 |

## 六、数据字段说明（CareLog）

| 字段 | 类型 | 说明 | 展示位置 |
|:---|:---|:---|:---|
| `id` | Long | 日志主键 | 列表 key |
| `logDate` | LocalDate | 记录日期 | 左侧主日期（如「2024年6月7日」） |
| `createdAt` | LocalDateTime | 创建时间 | 日期下方的具体时间（如「13:09」），同时用于同一天内排序 |
| `operationType` | String | 操作类型（浇水/施肥/修剪/换盆/其他） | 节点图标 + 类型标签 |
| `details` | String(TEXT) | 操作详情 | 卡片主体文字，支持展开/收起 |
| `growthStatus` | String(200) | 长势描述 | 绿色状态标签 🌱 |
| `diseaseStatus` | String(200) | 病害情况 | 橙色状态标签 ⚠️ |
| `imageUrls` | String(TEXT) | 图片 URL 集合（支持 JSON 数组或逗号分隔字符串） | 底部图片网格 |

## 八、修复记录

| 问题 | 原因 | 修复方案 |
|:---|:---|:---|
| 滚动到末尾不加载下一页 | `sentinelRef` 位于 `v-else` 块内，初始 logs 为空时该元素不渲染，`onMounted` 中 observer 无法绑定 | 1. 将 sentinel 移出条件块，始终渲染在 DOM 中；2. 提取 `setupObserver()` 独立方法；3. 在 `onMounted`、`watch(logs)`、`resetAndLoad()` 中配合 `nextTick()` 多次确保 observer 绑定正确；4. observer 每次绑定时先断开旧实例避免重复监听 |
| 同一天内记录顺序错误 | Repository 只按 `logDate`（LocalDate，只有日期）倒序，同一天内缺少二级排序 | 将 Repository 方法从 `findByPlantArchiveIdOrderByLogDateDesc` 更名为 `findByPlantArchiveIdOrderByLogDateDescCreatedAtDesc`，新增 `createdAt`（LocalDateTime）作为二级倒序排序字段；同步更新 Service 层所有调用处 |
